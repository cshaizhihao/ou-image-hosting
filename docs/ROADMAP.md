# OU-Image Hosting 十轮开发路线图

## 总体规则

每轮结束必须：

1. 完成功能、测试和视觉检查。
2. 更新版本号、README、CHANGELOG 和截图。
3. 执行项目名称与代码来源检查。
4. 提交并推送 GitHub `main`。
5. 输出 Commit Hash、测试结果和下一轮风险。

## Round 1 — v0.1.0 品牌与工程基线

- 确定产品定位、用户、品牌语气和视觉方向。
- 建立全新 GitHub 仓库。
- 收录 Logo，建立品牌资产目录。
- 编写主 Prompt、十轮路线图、README、CHANGELOG。
- 确定原创技术架构和目录结构。
- 定义版本与仓库同步规范。

## Round 2 — v0.2.0 设计系统与应用壳层

- 初始化 pnpm monorepo。
- 创建 Web、API、UI、Shared packages。
- 落地 Design Tokens、浅色和深色主题。
- 构建侧栏、顶栏、命令面板、Toast、Dialog、Drawer。
- 完成登录、注册、找回密码的高完成度界面。
- 生成桌面、移动、浅色、深色 README 截图。

## Round 3 — v0.3.0 上传引擎

- 文件选择、拖拽、剪贴板、URL 上传。
- 多文件队列、逐文件状态、总进度。
- 暂停、取消、失败重试和重复文件检测。
- 文件类型、大小、尺寸和配额校验。
- Sharp 元数据读取、缩略图和压缩。
- 上传完成后复制 URL、Markdown、HTML、BBCode。

## Round 4 — v0.4.0 图片库

- 响应式瀑布流、等宽网格和列表视图。
- 文件名、标签、相册、时间和存储筛选。
- 多选、全选、批量标签、移动、删除和下载。
- 虚拟滚动、懒加载和骨架屏。
- 空状态、错误状态和恢复流程。
- 保存筛选条件与滚动位置。

## Round 5 — v0.5.0 图片详情与编辑

- 图片详情 Drawer 与独立详情页。
- 元数据、尺寸、大小、格式、上传者和访问数据。
- 裁剪、旋转、格式转换、质量调整和水印。
- 版本历史、原图恢复和派生图管理。
- 分享链接、过期时间和密码保护。
- 回收站与撤销删除。

## Round 6 — v0.6.0 存储与链接

- 本地磁盘、S3、Cloudflare R2。
- 通用 S3-compatible 适配器。
- 存储桶健康检查和连通性测试。
- 默认存储、按规则路由和迁移任务。
- 自定义域名、链接模板、签名 URL。
- 防盗链、Referer 规则和缓存策略。

## Round 7 — v0.7.0 用户、团队与安全

- 注册、登录、Session、密码重置。
- 个人、管理员、成员和只读角色。
- 团队空间、邀请和成员管理。
- API Token、权限范围和吊销。
- 登录历史、审计日志和危险操作确认。
- 限流、CSRF、CSP、文件安全检查。

## Round 8 — v0.8.0 数据与可观测性

- 上传量、存储量、流量和请求趋势。
- 热门图片、格式分布、存储分布。
- 任务队列、失败率和处理耗时。
- 活动时间线和审计筛选。
- 健康检查、结构化日志和错误追踪接口。
- CSV 导出与图表无障碍摘要。

## Round 9 — v0.9.0 设置、部署与文档

- 设置中心按领域拆分。
- 环境变量校验和安装向导。
- Dockerfile 与 Docker Compose。
- 数据库迁移、备份和恢复命令。
- 部署、安全、存储和升级文档。
- README 安装、使用、截图与常见问题完善。

## Round 10 — v1.0.0 正式发布

- 全量单元、集成、E2E 和视觉回归测试。
- 375、768、1024、1440 响应式检查。
- 浅色、深色、键盘、屏幕阅读器和 reduced-motion 检查。
- 性能预算、图片优化和大图库压力测试。
- 安全审计、依赖审计和威胁建模。
- 发布 v1.0.0、Git Tag、Release Notes 和最终截图。
