# OU-Image Hosting 十轮开发路线图

当前进度：**第 10 / 10 轮完成**

## 总体规则

每轮结束必须：

1. 完成功能、测试、构建和视觉检查。
2. 更新版本号、README、CHANGELOG 和真实应用截图。
3. 检查仓库中不存在外部项目名称、私人资料和临时参考文件。
4. 使用 Conventional Commits 提交并推送 GitHub `main`。
5. 输出版本号、Commit Hash、测试结果、当前轮次和下一轮目标。
6. CPU 密集型任务必须使用 `scripts/run-low-cpu.sh`，持续 CPU 使用率保持在 30% 以下。
