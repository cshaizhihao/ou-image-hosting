# OU-Image Hosting v1.0.8

`v1.0.8` 是安装与更新热修复版本，解决安装机执行 `ouih update` 时因为本地 `main` 和远端 `origin/main` 分叉而无法 fast-forward 的问题。

## 修复内容

- `ouih update` 不再使用 `git pull --ff-only origin main`。
- 安装器重复运行也不再依赖 fast-forward。
- 更新流程会先备份 `.env.production`，再把代码工作树安全对齐到远端 `origin/main`，最后恢复生产配置。
- `.env.production`、`OU_SECRET_KEY`、域名、端口、Cloudflare/Caddy 模式和数据卷不会被清空。
- 如果安装目录确实有未提交源码改动，仍会拒绝覆盖，避免误删手工改动。

## 新的同步流程

```text
备份 .env.production
git fetch --depth 1 origin main
git checkout -B main origin/main
git reset --hard origin/main
恢复 .env.production
顺序构建 API / Web
启动服务
```

## 已安装用户如何更新

如果你的旧版 `ouih update` 已经卡在 `fatal: Not possible to fast-forward, aborting.`，请先执行一次：

```bash
curl -fsSL https://raw.githubusercontent.com/cshaizhihao/ou-image-hosting/main/install.sh | bash
```

这会使用新版安装器修复安装目录，保留生产配置并升级到最新版本。之后日常更新继续使用：

```bash
ouih update
```
