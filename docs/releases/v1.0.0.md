# OU-Image Hosting v1.0.0

OU-Image Hosting / 欧记图床完成十轮开发并发布首个稳定版本。

## 主要能力

- 现代响应式图床工作台：安装、认证、批量上传、图库、图片处理、版本、分享、相册、标签、收藏和回收站。
- 团队与安全：多工作区 RBAC、API Token、TOTP、会话、通知、审计和来源/IP 边界。
- 运维与可观测：数据统计、系统状态、健康探针、后台任务、备份恢复、设置中心和性能预算。
- 生产部署：Docker Compose 双容器、运行时 API 代理、非 root 运行、只读文件系统、资源配额和 GitHub Actions。
- 品牌体验：原始猫咪 Logo 保持不变，接入 Swei Ax Sans CJK SC Black 授权网页子集，完整支持浅色、深色和移动端。

## 快速部署

```bash
git clone https://github.com/cshaizhihao/ou-image-hosting.git
cd ou-image-hosting
cp .env.production.example .env.production
openssl rand -hex 32
```

将随机值写入 `OU_SECRET_KEY`，设置实际 HTTPS `APP_ORIGIN`，然后：

```bash
COMPOSE_PARALLEL_LIMIT=1 docker compose --env-file .env.production build
docker compose --env-file .env.production up -d
curl --fail http://127.0.0.1:3000/api/health/ready
```

## 发布验证

- 40 个单元与集成测试。
- Playwright 安装、认证、上传、分享、管理和键盘流程。
- Axe serious/critical 无障碍问题为零。
- 375、768、1024、1440 宽度，浅色、深色和 reduced-motion 回归。
- 生产依赖审计无已知 moderate 及以上漏洞。
- Shared First Load JS 103 kB，最大业务路由 166 kB。
- 本地生产 p95：live 2.0 ms、ready 5.0 ms、登录页 6.6 ms。

## 部署边界

- 当前权威元数据存储是单 API 进程管理的原子 JSON 文件，只运行一个 API 副本。
- 当前权威图片存储是本地持久化卷；S3/R2 支持配置、探测和迁移，但尚未切换日常读写。
- PostgreSQL、Redis 与 CDN 环境变量用于状态探测，不代表业务已切换。
- 正式环境必须使用 HTTPS 反向代理、随机 `OU_SECRET_KEY` 和站外备份。
