# Changelog

本项目遵循 Semantic Versioning。

## [0.8.0] - 2026-07-10

### Added

- 新增独立存储与分发控制台，提供桌面、移动端和深色主题完整布局。
- 新增本地存储容量、文件数量、文件系统空间和真实读写连接测试。
- 新增 Amazon S3、Cloudflare R2 与 S3-compatible 配置和 AWS SigV4 Bucket 探测。
- 新增本地到 S3/R2 的逐文件迁移、状态、完成数、失败数和错误记录。
- 新增自定义域名、链接模板、Referer 白名单、空 Referer 策略和签名 URL。
- 新增 gzip 完整备份、下载、恢复、删除、保留数量和 SHA-256 清单校验。
- 新增存储概览、防盗链与签名、备份管理和移动端深色真实截图。
- 新增 schema v4 到 v5 迁移和存储基础设施集成测试。

### Changed

- 持久化结构升级到 schema v5，增加存储、交付、备份和迁移状态。
- 图片库、详情、版本、收藏、相册、标签和回收站统一生成交付 URL。
- 远端凭据不再回显；秘密字段留空时继续使用已保存密钥。
- 当前活动读写源固定为本地存储，S3/R2 仅在完成读写闭环后才允许激活。
- 删除态图片的当前文件和历史版本文件统一返回 404。
- 自动化测试增加到 21 个，并完成真实签名链接、防盗链拒绝、备份下载恢复和响应式浏览器验证。
- 十轮路线图推进至第 7 / 10 轮。

### Security

- S3/R2 Secret Access Key 使用 `OU_SECRET_KEY` 派生的 AES-256-GCM 密钥加密保存。
- 签名链接使用 HMAC-SHA256、明确过期时间和常量时间签名比较。
- 防盗链按规范化 HTTP/HTTPS Origin 校验，不接受任意 Referer 字符串匹配。
- 链接模板限制协议和变量，拒绝缺少图片 ID 或文件变体的模板。
- 备份恢复验证格式、schema、状态摘要、路径、大小和每个文件的 SHA-256。
- 存储、分发、迁移和备份接口仅允许站点 Owner 使用。

## [0.7.0] - 2026-07-10

### Added

- 新增相册创建、编辑、描述、封面、删除和相册图片聚合接口与页面。
- 新增带颜色的标签创建、编辑、删除、标签图片聚合和跨标签合并。
- 新增图片详情页收藏开关、相册归属和多标签选择组件。
- 新增独立收藏页面，支持集中查看并快速移出收藏。
- 新增回收站页面、批量选择、恢复和永久删除确认流程。
- 新增相册、标签、图片整理、收藏、回收站与移动端深色真实截图。
- 新增 v3 到 v4 数据迁移、关系隔离、非法关系 ID 和物理文件清理测试。

### Changed

- 持久化结构升级到 schema v4，为图片增加 `favorite`、`albumIds` 和 `tagIds`。
- 图片列表和聚合响应统一携带组织字段、更新时间和删除时间。
- 图片详情响应增加收藏、相册和标签关系。
- 上传列表、汇总、去重和批量回收按当前用户严格隔离。
- 永久删除会清理所有版本文件、分享记录和相册封面引用，并保护仍被引用的物理文件。
- 前端组织更新采用轻量响应增量合并，避免覆盖版本历史和分享数据。
- 回收站操作兼容 `restored`、`deleted` 与旧 `updated` 计数字段。
- 自动化测试增加到 17 个，并完成相册、标签合并、收藏、封面、恢复、永久删除和移动端浏览器验证。
- 十轮路线图推进至第 6 / 10 轮。

### Security

- 相册、标签、收藏和回收站接口仅允许访问当前用户拥有的数据。
- 图片组织关系严格验证相册和标签 ID 的所有权与存在性。
- 永久删除使用受限存储根目录解析，拒绝越界存储键。
- 物理文件删除对所有版本去重，容忍文件已缺失，同时避免删除其他图片仍引用的文件。

## [0.6.0] - 2026-07-10

### Added

- 新增独立图片详情页，展示当前图片元数据、SHA-256、原图链接和版本状态。
- 新增 URL、Markdown、HTML 与 BBCode 四种引用格式一键复制。
- 新增安全重命名接口与前端编辑表单。
- 新增 Sharp 左右旋转、水平/垂直翻转、JPEG/PNG/WebP 转换和质量控制。
- 新增图片版本链、历史版本文件访问和不破坏当前记录的版本恢复。
- 新增带可选密码、有效期、撤销状态和访问统计的分享链接。
- 新增公开分享元数据与密码验证后图片二进制访问接口。
- 新增本地二维码生成、公开分享查看/下载页和公开路由放行。
- 新增图片详情、密码分享、公开查看和移动端深色真实截图。

### Changed

- 持久化结构升级到 schema v3，并自动把旧图片迁移为原始版本记录。
- 存储使用量改为统计所有版本中唯一物理原图文件，恢复引用不重复计费。
- 图片变换在写盘前检查真实物理配额，派生版本不能绕过容量限制。
- 当前原图缓存策略改为短期重新验证，避免变换后继续显示旧版本。
- 图片库预览入口改为进入详情页，单独复选框继续负责批量选择。
- 自动化测试增加到 15 个，并完成编辑、恢复、密码分享、访问统计、撤销和移动端浏览器验证。
- 十轮路线图推进至第 5 / 10 轮。

### Security

- 分享令牌只存 SHA-256 摘要，分享密码只存 scrypt 哈希。
- 图片详情、编辑、版本恢复和分享管理仅允许图片所有者访问。
- 重命名拒绝路径分隔符、控制字符和 `.`、`..` 特殊名称。
- 公开分享响应不暴露密码哈希、令牌哈希、存储键或文件系统路径。
- 分享访问对过期、撤销和错误密码返回明确受控状态并启用限流。

### Added

- 新增图片名称搜索、格式筛选和最新、最早、名称、大小排序。
- 新增服务端分页参数与统一分页响应，单页最多返回 100 张图片。
- 新增单张选择、本页全选、确认对话框和批量移入回收站接口。
- 新增视图、筛选、排序偏好与滚动位置的浏览器端恢复。
- 新增加载骨架、无结果、空图库、请求失败和重试状态。
- 新增桌面网格、列表选中态和移动端深色图片库真实截图。
- 新增图库筛选、排序、分页与批量回收集成测试。

### Changed

- 上传汇总与图片列表默认排除已进入回收站的资产。
- 重复上传已回收图片时自动恢复该资产，而不是创建重复记录。
- 专用 `/library` 页面从动态占位路由中独立，避免预生成路径冲突。
- 自动化测试增加到 11 个，并完成登录、视图切换、批量回收、搜索、筛选和偏好持久化浏览器验证。
- 十轮路线图推进至第 4 / 10 轮。

### Security

- 图库查询对分页大小、格式和排序字段使用白名单约束。
- 批量回收操作限制为单次最多 100 个图片 ID。
- 回收操作只更新受当前工作区管理的已存在图片记录。

## [0.4.0] - 2026-07-10

### Added

- 新增 multipart 本地图片上传 API，单文件上限 20 MB。
- 新增 JPG、PNG、WebP、GIF 与 AVIF 服务端内容识别和 8000 万像素限制。
- 新增 Sharp 元数据读取、EXIF 自动旋转与 640px WebP 缩略图生成。
- 新增 SHA-256 内容去重，重复图片直接复用已有资产与链接。
- 新增原图、缩略图文件服务和不可变原图缓存策略。
- 新增 2 GB 默认存储配额、使用汇总和侧栏容量进度。
- 新增 URL 图片上传与 DNS 私网、回环、链路本地地址拦截。
- 新增本地选择、拖拽、全局剪贴板粘贴和 URL 上传入口。
- 新增串行上传队列、逐文件/总进度、暂停、恢复、取消和失败重试。
- 新增重复文件标记、真实缩略图、尺寸信息和绝对原图链接复制。
- 新增桌面、移动深色模式和 URL 上传对话框真实截图。

### Changed

- 持久化结构升级到 schema v2，并自动迁移 v0.3.0 数据。
- 上传工作台由浏览器本地预览升级为真实本地存储上传。
- 用户菜单侧栏展示真实已用容量与总配额。
- 自动化测试增加到 10 个，覆盖真实 PNG 上传、缩略图、去重和 SSRF 拦截。
- 十轮路线图推进至第 3 / 10 轮。

### Security

- 服务端根据图片内容而非客户端 MIME 决定文件类型。
- 原图与缩略图使用服务端生成键名，上传文件名不参与磁盘路径。
- URL 上传限制协议、下载时长、响应大小并拒绝受保护网络地址。

## [0.3.0] - 2026-07-10

### Added

- 新增三步首次安装向导、Node.js 与数据目录环境检查。
- 新增 JSON 文件持久化仓库，用于站点配置、用户、会话与密码重置记录。
- 新增管理员创建、开放注册开关、登录、注册、退出和会话查询接口。
- 新增 `scrypt` 密码散列、会话令牌摘要、登录失败锁定和接口限流。
- 新增找回密码与一次性密码重置流程，重置后自动撤销全部旧会话。
- 新增密码强度提示、首次使用引导和浅色、深色、跟随系统主题偏好。
- 新增认证路由保护、真实用户菜单与退出操作。
- 新增 6 组 API 集成测试，共覆盖安装、校验、认证、会话、重置与防枚举行为。
- 新增安装、配置、首次引导、已登录工作区和移动端深色登录截图。

### Security

- 会话使用 `HttpOnly`、`SameSite=Lax` Cookie，生产环境默认启用 `Secure`。
- 服务端仅保存会话与密码重置令牌的 SHA-256 摘要。
- 认证响应禁止缓存，敏感请求字段与 Cookie 在日志中自动脱敏。
- 写请求校验来源域，登录与密码重置接口启用独立限流。
- 密码找回响应不暴露账号是否存在。

### Changed

- Web 通过同源 `/api/*` 代理访问 Fastify，避免浏览器端暴露内部 API 地址。
- README 更新 v0.3.0 安装、初始化、认证流程、安全配置与真实截图。
- 十轮路线图推进至第 2 / 10 轮。
