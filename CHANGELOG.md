# Changelog

本项目遵循 Semantic Versioning。

## [1.0.7] - 2026-07-11

### Added

- 新增独立工作区概况面板，展示真实图片数量、存储占用、容量比例、工作区角色和快捷操作。
- Amazon S3 与 Cloudflare R2 卡片新增分步配置教程，支持上一步、下一步、方向键、Esc、步骤进度以及官方控制台和文档快捷链接。
- 新增存储教程数据模型、官方链接约束测试、概况模型测试和壳层会话快照测试。

### Fixed

- 修复页面切换或工作区会话重新载入时，管理与系统导航分组短暂消失的问题。
- 概况路由不再落入通用空状态页面，站点所有者可直接进入存储配置。
- 修复宽屏桌面 100% 缩放下导航、菜单和主要操作过小且主内容区域留白过多的问题。

### Changed

- 桌面侧栏调整为 272px、顶栏调整为 72px、正文基准调整为 16px，主内容最大宽度提高到 1480px。
- 存储、管理、图片详情与组织页面同步放宽，并提高高频说明文字和状态信息的可读字号。
- 高频导航、卡片和按钮取消多余位移/缩放；抽屉与弹窗统一为更短的透明度和小幅位移动画。
- 上传、存储迁移、系统任务、分析与概况进度条改用 `transform: scaleX()`，减少持续状态更新造成的布局重排。

### Tests

- 全量 `pnpm check` 通过：lint、类型检查、52 个单元测试、安装器、`ouih` 与生产构建。
- Playwright 7/7 通过，包括核心业务、命令面板、Axe 可访问性、多断点无横向溢出和 `prefers-reduced-motion`。

## [1.0.4] - 2026-07-11

### Added

- 安装器新增系统包管理器识别，支持 apt、dnf、yum、pacman、zypper、apk 与 Homebrew。
- 缺少 Git、curl、OpenSSL、coreutils 或 CA 证书时，安装器会列出缺失项并自动安装。
- Linux 缺少 Docker Engine 或 Docker Compose v2 时，apt/dnf/yum 使用 Docker 官方安装程序，pacman/zypper/apk 使用发行版原生软件包，并尝试启动 Docker 服务。
- 新增 `--no-install-deps`，供自行维护系统依赖的用户禁止自动安装。

### Changed

- 环境检查从“发现缺失立即退出”改为“检测、确认、安装、复检”完整流程。
- `--yes` 无人值守模式会自动补齐依赖；交互模式默认询问确认。
- Arch Linux 的依赖安装执行完整系统升级，避免 `pacman -Sy` 导致部分升级。

### Tests

- 安装器测试新增 apt 基础依赖自举、命令行禁用自动安装、Arch 安全升级、Docker 官方脚本 TLS 下载、Compose 修复、服务启动与原生 Docker 软件包模拟场景。

- 现有配置改为就地更新指定键并保留未知键，不再通过重写完整文件清空部署扩展参数。
- `--no-start` 和安装完成页统一使用 `ouih start/status/logs/stop`，确保 Caddy 与 Cloudflare 模式始终带上 `https` profile。

### Tests

- 安装器集成测试新增重复无人值守升级配置保留、未知变量保留、Cloudflare 200 成功、403/525/526 拒绝、缺失 CF 标识拒绝和 HTTPS 延迟启动场景。

## [1.0.2] - 2026-07-11

### Added

- 新增 Compose 内置 Caddy HTTPS 反向代理，通过 `https` profile 自动申请、续期并持久化公网域名证书。
- 新增全局 `ouih` 管理命令与品牌化交互菜单，集中提供更新、卸载、访问地址、安装目录、状态、日志和启停操作。
- 新增 `--proxy caddy|cloudflare|external|none` 安装参数，兼容域名直连、Cloudflare 小黄云、已有反向代理和本机 HTTP 四种部署方式。
- 新增 Caddy 证书数据卷、配置卷、健康检查与安装后 HTTPS 入口验证。

### Changed

- HTTPS 域名安装默认启用 Caddy，并将 Web 服务固定绑定到 `127.0.0.1`，不再把反向代理留给用户手工配置。
- 安装器会预检宿主机 80/443 端口、等待本地 readiness，并进一步验证真实证书与 HTTPS 反向代理链路。
- Cloudflare 模式会继续验证边缘响应中的 `CF-Ray` / Cloudflare 标识，确认小黄云链路真正生效。
- README 快速开始、日常运维、Cloudflare DNS 指引和故障排查统一改为自动 HTTPS 与 `ouih` 工作流。

### Fixed

- 修复安装器在没有 TLS 终止层时仍输出 HTTPS 安装成功，导致浏览器出现 `ERR_SSL_PROTOCOL_ERROR` 的问题。
- 修复公网域名配置正确但 Compose 仅暴露 HTTP 回环端口，用户无法直接通过域名访问的问题。
- 更新与卸载流程默认保留生产配置、加密密钥和 Docker 数据卷，危险清理必须显式确认。

## [1.0.0] - 2026-07-11

### Added

- 新增 Docker 双容器生产部署、内部 API 网络、回环端口绑定、只读根文件系统、非 root 运行、健康检查、日志轮转和合计低于 0.30 CPU 的运行配额。
- 新增 GitHub Actions 质量检查与多架构容器镜像流程，以及部署、备份恢复、升级、故障排查、威胁模型和性能预算文档。
- 新增 `/health/live` 与执行真实写入、重命名、读取和删除探针的 `/health/ready`，恢复维护期间 readiness 返回 503。
- 新增备份恢复维护门、活动写请求排空、严格归档大小/文件数/压缩比限制、同卷 staging、原子目录切换和失败回滚。
- 新增 Playwright 核心流程、Axe 无障碍、四档响应式、浅色/深色、reduced-motion 与最终截图自动化。
- 新增顺序 HTTP 性能冒烟脚本和生产路由 JavaScript 体积预算。
- 新增 Swei Ax Sans CJK SC Black 的 155,880 字节授权 WOFF2 网页子集、SIL OFL 1.1 文本和字体来源说明。
- 新增 v1.0.0 桌面上传、命令面板、设置中心和移动端深色真实截图。

### Changed

- Web `/api/*` 从构建期固化 rewrite 改为 Node Route Handler 运行时流式同源代理，`API_PROXY_TARGET` 修改后无需重建 Web。
- 生产 Web 配置从 `next.config.ts` 改为 `next.config.mjs`，避免精简生产依赖启动时尝试安装 TypeScript。
- 未登录跳转根据受信转发主机或当前 Host 构造同源绝对地址，兼容 HTTPS 反向代理与本地生产启动。
- 生产环境启动时强制校验随机 `OU_SECRET_KEY`、有效 `APP_ORIGIN`、HTTPS 边界和关闭开发重置令牌。
- API 支持 SIGTERM/SIGINT 30 秒优雅关闭；状态持久化成功后才更新进程内状态。
- 十轮路线图完成第 10 / 10 轮，仓库、运行时接口与包版本统一为 `1.0.0`。

### Fixed

- 修复备份恢复可能与并发写请求交错、失败后留下部分新状态或覆盖可回滚目录的问题。
- 修复恶意或异常归档可通过路径、重复条目、base64、校验和、文件数量、解压大小或压缩比消耗资源的问题。
- 修复 Web 生产部署的 API 地址在构建阶段被固定，导致 Docker 运行时 `API_PROXY_TARGET` 不生效的问题。
- 修复 Next.js 生产中间件使用错误主机或相对 Location 时造成跨域跳转与 `Invalid URL` 的问题。
- 修复上传区域键盘交互、命令面板焦点可见性和部分浅色小字号标签对比度不足的问题。

### Security

- 生产依赖审计无已知 moderate 及以上漏洞，PostCSS 锁定到已修复版本。
- 容器丢弃全部 Linux capabilities、启用 `no-new-privileges`、限制 PID/内存/CPU，并仅让 API 暴露在内部网络。
- 备份恢复拒绝越界路径、重复条目、非法状态、超限文件和不匹配 SHA-256，失败时保持旧数据可恢复。
- API 代理移除 hop-by-hop 头、禁用上游压缩歧义并保留同源 Cookie 与查询参数。
- 最终 E2E 覆盖安装、认证、上传、分享、管理、键盘操作、无障碍与响应式状态。

- 新增原生 SVG 趋势图、读屏数据表，以及上传与分享访问分别独立的数据覆盖范围提示。
- 新增站点 Owner 专属系统状态页、最近状态快照、按需主动检查、服务架构模式和系统检查时间线。
- 新增备份与存储迁移统一后台任务视图、进度展示、失败代码和受控失败任务重试。
- 新增站点公开信息、上传上限、允许格式、处理质量、缩略图宽度、语言、时区和默认外观设置。
- 新增全局真实离线横幅，以及数据统计、系统状态、站点与处理设置和移动端深色真实截图。
- 新增 `DATABASE_URL`、`REDIS_URL` 与 `CDN_BASE_URL` 空配置示例，用于系统状态中的可选服务配置探测。

### Changed

- 系统状态读取与主动检查分离：`GET /system/status` 只读取最近结果，`POST /system/status/check` 才执行真实检查。
- 数据统计不再使用虚构带宽或泛化浏览量，趋势只基于按日记录的上传和真实分享访问事件。

- PostgreSQL、Redis 与 CDN 按“未配置”“已配置但当前未使用”或真实探测状态呈现，不把配置存在解释为业务已切换。
- 工作区运行设置所有成员可读，仅 Admin/Owner 可保存；站点公开设置仅站点 Owner 可读写。
- 十轮路线图推进至第 9 / 10 轮，仓库版本统一为 `1.0.0-rc.1`。

### Fixed

- 修复历史累计分享访问可能被误解为每日趋势的问题，未归因访问不会被填入时间序列。
- 修复上传历史覆盖范围不明确的问题，数据不完整时会显示真实追踪起始日期。
- 修复分析、系统状态和站点设置响应缺少必填字段时被默认值或未知状态静默掩盖的问题。
- 修复未配置 PostgreSQL、Redis、CDN 被误报为系统故障的问题。
- 修复网络断开时缺少全局反馈的问题，并保留恢复联网后的操作提示。

### Security

- 系统状态、全站任务和任务重试仅允许站点 Owner 访问，前端导航和 API 同时执行权限限制。
- 站点设置仅站点 Owner 可修改，工作区上传、处理、本地化与外观设置仅 Admin/Owner 可修改。
- 分析、系统状态、任务和设置公共 DTO 采用严格运行时校验，拒绝缺失或类型不符的数据。
- 主动系统检查和失败任务重试使用独立限流，并保持审计记录。
- 系统状态只暴露公开服务模式、探测结果和中性配置状态，不返回数据库或外部服务凭据。

## [0.9.0] - 2026-07-11

### Added

- 新增多工作区模型以及 Owner、Admin、Editor、Viewer 四级 RBAC。
- 新增团队邀请、邀请接受、角色调整、成员移除和工作区所有权转移。
- 新增八种精确 Scope、有效期、吊销和 IPv4/IPv6/CIDR 白名单的 API Token。
- 新增会话设备管理、TOTP 双因素认证、恢复代码和两阶段登录挑战。
- 新增安全、协作和系统三类站内通知、分类偏好、已读状态与免打扰时段。
- 新增工作区审计时间线、条件筛选和 Admin/Owner CSV 导出。
- 新增团队、邀请、API Token、安全设置、通知、审计和移动端深色真实截图。
- 新增 schema v5 到 v6 迁移、旧版备份恢复和安全边界集成测试。

### Changed

- 图片、上传、详情、版本、分享、相册、标签、收藏、回收站和基础设施路由统一按工作区与 Capability 授权。
- API Token 固定绑定创建时的工作区，不能访问团队、安全和其他管理接口。
- 导航与管理入口按当前工作区角色动态展示，未授权页面返回明确权限状态。
- 站内通知响应仅返回展示必需字段，并按当前工作区和通知偏好隔离。
- 自动化测试增加到 32 个，并完成团队、Token、2FA、通知、CSV 导出和响应式浏览器验证。
- 十轮路线图推进至第 8 / 10 轮。

### Security

- Cookie 写请求统一校验来源；API Token 采用严格 Bearer 解析与精确 Scope 校验。
- Token IP 白名单支持规范化 IPv4、IPv6、mapped IPv6 和 CIDR，来源不匹配时明确拒绝。
- 仅在配置可信代理地址后启用转发 IP，禁止无边界信任代理头。
- TOTP 登录挑战限制有效期和尝试次数，防止重放；恢复代码采用原子消费。
- 审计元数据经过字段过滤，来源 IP 使用 HMAC 摘要，日志 URL 移除敏感查询参数。
- 通知读取状态保持工作区隔离，其他工作区或无效事件 ID 不会写入用户状态。
- CSV 导出转义引号、逗号和换行，并阻断表格公式注入。
- v5 备份恢复经过结构归一化后升级为 schema v6，不绕过当前安全约束。

- 十轮路线图推进至第 7 / 10 轮。

### Security

- S3/R2 Secret Access Key 使用 `OU_SECRET_KEY` 派生的 AES-256-GCM 密钥加密保存。
- 签名链接使用 HMAC-SHA256、明确过期时间和常量时间签名比较。
- 防盗链按规范化 HTTP/HTTPS Origin 校验，不接受任意 Referer 字符串匹配。
- 链接模板限制协议和变量，拒绝缺少图片 ID 或文件变体的模板。
- 备份恢复验证格式、schema、状态摘要、路径、大小和每个文件的 SHA-256。
- 存储、分发、迁移和备份接口仅允许站点 Owner 使用。
